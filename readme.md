# **Mountain**

> PublicInsight 플랫폼 개발을 위한 기술 검증 및 아키텍처 설계를 위한 Full-Stack 파일럿 프로젝트

## **Overview**

본 프로젝트는 대규모 공공 데이터 플랫폼 'PublicInsight'의 성공적인 개발을 위한 PoC를 목적으로 기획됨
실제 운영 가능한 웹 애플리케이션을 구현하며 핵심 기술 스택의 적합성을 평가하고 안정적인 아키텍처 설계의 기반을 마련하고자 함

#### 프로젝트의 주요 목표는 다음과 같음.

*   **핵심 기술 스택 검증**: `Next.js (React)`, `FastAPI`, `MySQL` 조합의 Full-Stack 환경을 직접 구축하고 운영하며 기술적 특성과 안정성을 파악
*   **아키텍처 설계 경험**: 프론트엔드와 백엔드를 명확히 분리하고, `RESTful API`를 설계하여 두 영역 간의 데이터 통신 파이프라인을 구축
*   **데이터베이스 관리**: `SQLAlchemy ORM`을 활용한 모델링 및 `Alembic`을 통한 데이터베이스 스키마 버전 관리(마이그레이션) 경험을 확보
*   **웹 애플리케이션의 기본 구조 이해**: `JWT` 기반 인증/인가, 사용자-관리자 역할 분리, CRUD 기능 등 웹 서비스의 필수적인 구성 요소를 직접 구현하며 전체 개발 사이클에 대한 이해도를 심화

## **Key Features**

### **사용자 및 인증**

*   **회원 관리**: 회원가입, 로그인, 로그아웃, 프로필 조회 및 수정 기능 제공
*   **인증 시스템**: `JWT(JSON Web Token)`을 사용한 토큰 기반 인증 시스템 구현
*   **보안**: `Bcrypt` 라이브러리를 활용한 사용자 비밀번호 단방향 암호화

### **커뮤니티 (게시글 및 댓글)**

*   **CRUD**: 게시글 및 댓글의 생성(Create), 조회(Read), 수정(Update), 삭제(Delete) 기능 구현
*   **사용자 상호작용**: 게시글 및 댓글에 대한 '좋아요/싫어요' 기능
*   **콘텐츠 관리**: 부적절한 게시글 및 댓글에 대한 신고 기능
    *   설정된 신고 임계값 초과 시 해당 콘텐츠 자동 숨김 처리
    *   신고 접수 시 관리자에게 실시간 알림 전송

### **관리자**

*   **콘텐츠 관리**: 공지사항 작성 및 수정/삭제, 모든 게시글 및 댓글 강제 삭제
*   **사용자 관리**: 전체 사용자 목록 조회 및 특정 사용자 제재(활동 정지) 기능
*   **시스템 관리**: 게시판 카테고리 및 기관 정보 관리

## **Stack**

### **Frontend**

*   **Language**: TypeScript
*   **Framework**: Next.js (v13+), React
*   **Styling**: Tailwind CSS, shadcn/ui
*   **State Management**: React Context API

### **Backend**

*   **Language**: Python (3.11)
*   **Framework**: FastAPI
*   **Database ORM**: SQLAlchemy
*   **DB Migration**: Alembic
*   **ASGI Server**: Uvicorn

### **Database**

*   MySQL

### **Development Tools**

*   **IDE**: Visual Studio Code
*   **Version Control**: Git, GitHub

## **프로젝트를 통해 배운 점**
의지만 있는 진행이 얼마나 위험한지에 대해서 알게됨...
다른 사람들이 다 하는데는 이유가 있구나
### **소프트웨어 아키텍처 설계의 중요성**
**계층형 아키텍처의 필요성**:
* 단일 라우터 레이어에 모든 비즈니스 로직을 구현하는 방식은 프로젝트의 복잡도가 증가함에 따라 유지보수성과 데이터 무결성을 저해하는 한계를 명확히 인지함
* 향후 프로젝트에서는 Service, Repository 등 역할에 따라 계층을 명확히 분리하여 확장성과 안정성을 확보해야 함을 알게됨

**미들웨어의 역할**: 
* 인증/인가, 로깅 등 여러 엔드포인트에 걸쳐 반복적으로 필요한 기능을 처리하기 위한 미들웨어의 체계적인 설계가 코드 중복을 줄이고 애플리케이션 구조를 단순화하는 데 얼마나 중요한지 확인함

### **코드 품질 및 유지보수성 향상**
**일관된 코드 패턴의 가치**: 
* 정해진 규칙 없이 개발을 진행할 경우 코드 가독성이 저하되고, 기능 추가 및 수정 시 예측 불가능한 사이드 이펙트를 유발할 수 있음을 경험함.
* 일관된 디자인 패턴을 적용하고, 반복되는 설정값이나 상수를 중앙에서 관리하는 것이 장기적인 프로젝트의 안정성에 필수적임을 깨달음.

### **개발 프로세스 및 방법론의 필요성**
**설계 우선 원칙(맞나?)**: 
* 명확한 사전 기획 및 설계 단계 없이 개발을 진행했을 때 발생하는 요구사항 불일치와 빈번한 코드 변경의 비효율성을 체감함.
* 데이터의 흐름과 모델 관계를 사전에 명확히 정의하는 것이 전체 개발 속도와 결과물의 품질을 높인다는 것을 인지함.

**유연한 스키마 관리**:
* 초기 설계가 완벽할 수 없다는 사실을 인정하고, 변화하는 요구사항에 맞춰 데이터베이스 스키마나 API 명세를 두려워하지 않고 개선해 나가는 과정이 더 나은 프로덕트를 만드는 길임을 경험함.

### **안정적인 운영을 위한 인프라 고려**
**개발 환경의 중요성 (Docker)**: 
* 로컬 환경에 직접 의존성을 설치하며 발생하는 문제 해결 과정의 비효율성을 직접적으로 경험함
* Docker와 같은 컨테이너 기술을 도입하여 개발-운영 환경을 일치시키는 것의 필요성을 절감함.
  
**시스템 가시성 확보(로그로그)**:
* 체계적인 로깅 시스템의 부재가 문제 발생 시 원인 분석을 얼마나 어렵게 만드는지 직접 경험함.
* 이는 추후 트래블슈팅과 시스템 모니터링의 기반이 된다는 점을 인지함.

**자동화와 최적화**:
* 스케줄러를 통한 반복 작업 자동화, 그리고 불필요한 컴퓨팅 리소스 낭비를 줄이기 위한 지속적인 코드 최적화가 서비스의 안정성과 확장성에 직접적인 영향을 미친다는 점을 확인함.
